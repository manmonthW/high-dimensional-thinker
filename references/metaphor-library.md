# 降维隐喻素材库

## 万能隐喻场景

### 1. 做菜 / 厨房

| 技术概念 | 隐喻 |
|----------|------|
| API | 餐厅的菜单，你不需要知道后厨怎么做，只要看菜单点菜 |
| 缓存 | 提前切好的配菜，做菜时直接用，不用现切 |
| 负载均衡 | 多个窗口同时打饭，人多就自动分流到人少的窗口 |
| 微服务 | 中央厨房拆成多个档口，川菜、粤菜各管各的 |
| 单例模式 | 厨房只有一个主厨，所有订单都找他 |
| 异步处理 | 点完菜先坐着等，服务员会叫号，不用站在窗口等 |
| 数据库事务 | 点套餐要么全上，要么全不上，不能只上一半 |
| 过拟合 | 只会做家里那几道菜，换个食材就不会了 |

## 隐喻设计原则

### 好隐喻的特征

1. **简单**：5岁小孩能听懂
2. **具象**：有画面感、能想象
3. **熟悉**：来自日常生活
4. **可延展**：能继续展开细节
5. **精准**：核心映射关系准确

### 坏隐喻的特征

1. **太抽象**：用一个抽象概念解释另一个
2. **太生僻**：需要额外解释隐喻本身
3. **映射错误**：关键属性对应不上
4. **过度延展**：硬凑的部分破坏整体

### 设计技巧

1. **先找核心属性**：这个概念最本质的特征是什么？
2. **在日常中找匹配**：什么日常事物有相同属性？
3. **检验可延展性**：能不能顺着这个隐喻继续解释？
4. **测试理解**：向外行人讲一遍，看他能不能复述

### 分布式系统

| 概念 | 隐喻 |
|------|------|
| 分布式 | 多个厨师同时做菜 |
| 一致性 | 所有菜单显示的价格必须一样 |
| 可用性 | 24小时营业，随时能点菜 |
| 分区容错 | 分店之间断网了，各自还能正常营业 |

| 最终一致性 | 各分店价格不同步没关系，最后会统一 |
| Raft/Paxos | 几个人投票选队长 |
